Sidlayout

När man designar hemsidor så vet man (nästan) aldrig vem som kommer kolla på den. Besökarna på sidan kan ha olika storlekar på sina skärmar, olika webbläsare osv. Fixerad websideslayout är enklast att utveckla till, alla element på sidan har specifika mått. Detta är dock inte alltid en bra sak! Om någon kommer som har en mindre skärm (t.ex. en smartphone) än vad man utvecklat sidan till kommer sidan sträcka sig utanför skärmen och man måste scrolla åt sidorna för att kunna se hela hemsidan. Det kan också bli problem att läsa texten eftersom den är specificerad för en stor skärm kommer den vara för liten att läsa för folk med små skärmar. Flytande webbsideslayout anpassar sig till webbläsarens storlek. Detta gör den m.h.a. att använda relativa mått (procent) istället för absoluta mått (pixlar). Även denna metod har nackdelar. Exempelvis brukar det uppstå problem på väldigt stora skärmar. Responsiv webbsideslayout fungerar delvis som den flytande layouten med att den också använder relativa mått istället för absoluta mått. Men förutom det så använder sig responsiva layouter av "media queries" för att skicka ut olika designs till webbläsare med olika fönsterbredd.

Förut använde man sig ofta av tabeller för att skapa sina hemsidor eftersom det gav en ganska stilig layout. Allt blev uppdelat i kolumner och rader. Ibland lade till och med tabeller i celler i andra tabeller. Som bieffekt blev det mycket extra html-kod som blev svår att underhålla. I CSS använder man istället taggar till allt och kan sedan enkelt välja ut specifika taggar och placera dom vart man vill på sidan.

Position

  • position - Man kan bestämma hur elementen på en sida ska renderas. Nyckelorden är static, absolute, fixed och relative. static gör så att elementen renderas i ordning, detta är standardvärdet. absolute gör att elementet placeras relativt till sin förälders placering. fixed gör att elementet renderas relativt till webbläsaren, effekten blir att elementet står still även fast man bläddrar på sidan. relative gör att elementet placeras relativt till sin egentliga placering.
  • top - Det här attributet (och de tre som kommer här nedan) har olika betydelser beroende på vilket värde position har. Men top bestämmer avstådet från toppen, vilken "topp" som menas bestäms av position.
  • left - Det här attributet har olika betydelser beroende på vilket värde position har. Men left bestämmer avstådet från vänsterkanten, vilken "vänsterkant" som menas bestäms av position.
  • bottom - Det här attributet har olika betydelser beroende på vilket värde position har. Men bottom bestämmer avstådet från botten, vilken "bott" som menas bestäms av position.
  • right - Det här attributet har olika betydelser beroende på vilket värde position har. Men right bestämmer avstådet från högerkanten, vilken "högerkant" som menas bestäms av position.
  • z-index - När man använder absolut positionering för att placera element kan dom hamna ovanpå varandra. För att bestämma vilket element som ska hamna överst använder man z-index.
  • float - Används för att ändra flödet på en sida. Man använder värdena left eller right för att få något att hålla sig till en sida.
  • clear - Används för att förbjuda ett flytande element. Man kan använda nyckelorden left, right och both.

Storlek

  • width - Anger bredden på ett element.
  • height - Anger höjden på ett element.
  • max-width - Anger maxbredder för ett element. Om man har angett en mått som t.ex. påverkas av skärmstorleken kan man specificera här så det måttet inte blir större än ett absolut mått.
  • max-height - Anger maxhöjden för ett element. Om man har angett en mått som t.ex. påverkas av skärmstorleken kan man specificera här så det måttet inte blir större än ett absolut mått.
  • min-width - Anger minimumbredder för ett element. Om man har angett en mått som t.ex. påverkas av skärmstorleken kan man specificera här så det måttet inte blir mindre än ett absolut mått.
  • min-height - Anger minimumhöjden för ett element. Om man har angett en mått som t.ex. påverkas av skärmstorleken kan man specificera här så det måttet inte blir mindre än ett absolut mått.
  • vertical-align - Detta påverkar hur ett element ska renderas vertikalt relativt till ett annat element. Exempel:

    Det här är en mening.

    I det här fallet har "en mening" stilen vertical-align: 75%;
